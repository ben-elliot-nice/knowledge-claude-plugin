## gcf-search — Search and Discoverability

Content structure directly determines how findable pages are — both within Expert's internal search and in external search engines.

### URL depth

- Hierarchy determines URL structure: every nesting level adds a path segment
- Deeply nested URLs (`/cat/subcat/guide/topic/howto`) harm external SEO
- Design the hierarchy to keep article URLs as short as practical
- Decouple page title from URL for categories with many sub-pages — a renamed category shouldn't break all child URLs

### Page titles

- Must be **unique and specific** across the knowledge base
- Most important keyword should appear **at the beginning** of the title
- Titles are the strongest SEO signal — they appear in search result headings

### Page summaries

- **Critical for external search engines** — summaries appear in search result snippets
- Must concisely describe what the page contains
- Do not leave summaries blank — blank summaries produce poor search snippets and reduce click-through

### Tags

- Add discoverability for search terms not already in the title, summary, or body
- Do not duplicate prominent terms already in the content — tags complement, not repeat
- 5-tag limit per page for optimal performance
- Identical tags across related pages strengthen article relationships and recommendation signals

### Classifications

- Appear as filter options in search results by default
- Work with Expert's internal search algorithm to improve result ordering
- Applying classifications consistently across a content set makes filtered search reliable

### Content formatting for findability

- Use ordered and unordered lists — lists improve findability within pages
- Add alt text and meaningful file names to images and media
- H1–H3 headings drive the auto-generated Table of Contents on Topic, How-To, and Reference pages

### Hierarchy design implications

The IA decisions made at structure time have lasting search consequences:
- Overly deep hierarchies create long URLs and dilute page authority
- Mixing organisational paradigms at the same level produces inconsistent navigation and weaker category-level SEO signals
- Generic, future-proof category names avoid URL churn when products or personas change
